TiltFiveGodot is GDNative extension for the Godot engine to connect to the Tilt Five system. It implements Godot's ARVRinterface and has GDNative class called TiltFiveManager for scripts to connect glasses and handle connection events.

Platforms

Currently only Windows 10/11 is supported because that is the only platform supported by Tilt Five. T5 linux support is supposed to come at some point in the future and support for that platform will revisited when it becomes available.

Installing

You can unzip the tilt-five.zip to your root project folder or use Godot's import button in the AssetLib tab.

Using the extension

After installing the asset you will need to add the TiltFiveManager class to the Autoload tab in project settings.

The Tilt Five extension uses the AR/VR nodes from Godot AR/VR primer â€” Godot Engine (stable) documentation in English However the initializtion is different.

In addons\tilt-five\scenes there are two scenes that show setup code and scene nodes for Tilt Five.
TiltFiveRig is a very minimal scene that sets up the camera and wand.
t5-scene is a little more advanced scene that shows off reading wand input.

Initialization is easy. The script first registers to receive signals from the TiltFiveManager and then starts the service. The two parameters to start_service are some sort of application id and a version number. 

func _ready():
    TiltFiveManager.connect("glasses_event", self, "on_glasses_event")
    if TiltFiveManager.start_service("com.mygame", "0.1.0"):
        print("service started")
    else:
        print("Failed to start service")

The glasses_event signal has two paramters a glasses id string and an event number which will be one of the following.

  • E_ADDED = 1 - Glasses are plugged into the computer and identified but not necessarily available to be used.
  • E_LOST = 2 - Previously identified glasses can no longer be found.
  • E_AVAILABLE = 3 - Your application can attempt to use these glasses. Use the TiltFiveManager.connect_glasses method.
  • E_UNAVAILABLE = 4 - The glasses are not available for use. This can happen after calling connect_glasses.
  • E_CONNECTED = 5 - You application has successfully obtained that glasses for exclusive use. The application can start rendering and the ARVRCamera and ARVRController nodes will start tracking.
  • E_DISCONNECTED = 6 - Normally in response to calling TiltFiveManager.disconnect_glasses however it can happen if there is some sort of connection problem to the glasses. If the disconnect happened due to a connection problem the system should try to reconnect and the application will receive another E_CONNECTED event.
  • E_TRACKING = 7 - The glasses can see the tracking markers on the board and the ARVRCamera node is being updated with pose data.
  • E_NOT_TRACKING = 8 - The glasses can't see the tracking markers on the board and the ARVRCamera node is not being updated with pose data.

Given these events a typical signal handler for the TiltFiveManager would look like this. 

var connected_glasses = null

func on_glasses_event(glasses_id, event_num):
    print("Event on glasses ", glasses_id, ", ", GlassesEvent.keys()[event_num-1])
    if connected_glasses and connected_glasses != glasses_id:
        return
    match  event_num:
        GlassesEvent.E_AVAILABLE:
            print("Trying to connect to glasses ", glasses_id)
            connected_glasses = glasses_id 
            TiltFiveManager.connect_glasses(glasses_id, "Godot Tilt Five Demo")
        GlassesEvent.E_UNAVAILABLE:
            connected_glasses = null
        GlassesEvent.E_CONNECTED:
            print("Glasses ", connected_glasses, " connected")
            arvr = true
        GlassesEvent.E_DISCONNECTED:
            connected_glasses = null
        GlassesEvent.E_TRACKING:
            print("Started tracking glasses ",glasses_id)
        GlassesEvent.E_NOT_TRACKING:
            print("Stopped tracking glasses ",glasses_id)
